# AI prompts & visual direction

Bushi generates two kinds of content with Cloudflare's AI stack: **text**
(promotional copy, recaps, the organizer assistant) via Workers AI text models,
and **imagery** (hero art, section illustrations, OG/result cards) via FLUX. This
doc captures the visual direction and the concrete prompts so the look stays
consistent and any asset can be regenerated deterministically.

Models (from the code + `@bushi/ai`):
- **Text:** `@cf/meta/llama-3.1-8b-instruct`
- **Embeddings:** `@cf/baai/bge-base-en-v1.5` (768-dim, → Vectorize)
- **Image:** `@cf/black-forest-labs/flux-1-schnell` (FLUX schnell)

---

## Visual direction

The aesthetic is **premium sports-tech editorial** — the feel of a modern
performance brand's launch page, not a dojo clip-art poster.

**Do**
- Restrained palette: near-black / graphite backgrounds, warm off-white type, a
  single disciplined accent (deep crimson `#B3122B` for the "red corner", cool
  indigo for the "blue corner"). Metallic/ink accents sparingly.
- Cinematic, low-key lighting; strong negative space; a clear focal subject.
- Texture and craft: matte surfaces, subtle grain, tatami weave, chalked hands,
  taped fists, folded gi fabric, a single mon/crest motif abstracted to a mark.
- Editorial composition — rule-of-thirds, generous margins, room for a headline.

**Don't**
- No clichés: no dragons, no flaming kanji, no neon katanas, no cartoon ninjas,
  no gradient "esports" swooshes.
- No literal text baked into generated images (headlines are composited by the
  layout/Browser Rendering layer, not the model).
- No busy backgrounds that fight overlaid UI.

**Recurring motifs:** the mat (tatami grid, seams), the corner (red/blue), the
crest (a single abstracted mark, never a full logo), disciplined athletic
silhouettes, motion frozen at the apex.

## Image prompts (FLUX)

Each entry is a ready-to-run prompt for `flux-1-schnell`. Keep a fixed negative
intent in the prompt itself (FLUX schnell doesn't take a separate negative field),
and pass a stable `seed` per asset so regenerations are reproducible.

### Hero (landing / marketing top)
```
Premium sports-tech editorial hero image for a martial arts tournament platform.
A lone disciplined athlete silhouette on a dark tatami mat, cinematic low-key
lighting, single warm rim light, deep graphite background, restrained palette
with one crimson accent. Matte textures, subtle film grain, vast negative space
on the right for a headline. Wide 16:9 composition, high detail, no text, no
logos, no clichés (no dragons, no neon, no flames).
```
Suggested: `width: 1600, height: 900`, store as `hero/landing-hero`.

### Section illustration — "Run the day" (command center)
```
Editorial illustration of a tournament command center: an overhead abstract of
multiple tatami mats arranged as a grid, warm off-white lines on graphite, a
single crimson highlighted mat, calm and precise, architectural, matte finish,
soft shadow, generous margins. No text, no logos, muted premium palette.
```

### Section illustration — "Live scoring"
```
Close, cinematic detail of taped fists and a chalked palm at the edge of a dark
competition mat, red-corner crimson accent, shallow depth of field, low-key
studio lighting, matte grain, editorial sports-tech mood. No text, no logos.
```

### Section illustration — "Discovery / schools"
```
Minimal editorial map-of-community motif: abstract constellation of small crest
marks connected by thin off-white lines over a graphite field, one indigo accent
node, calm and premium, lots of negative space. No text, no real logos.
```

### OG image background (per-tournament social share)
```
Clean 1200x630 social-share background for a martial arts event: dark graphite
gradient with a faint tatami weave texture, a single crimson corner accent lower
-left, strong empty central band for compositing an event title and date.
Premium, restrained, cinematic, no text, no logos.
```
Suggested: `width: 1200, height: 630`. The title/date/venue are composited on top
by `@bushi/rendering` (`ogImageHtml`), not generated by the model.

### Result card background (per-match / podium)
```
Vertical result-card background, 1080x1350, deep graphite with a subtle vertical
tatami seam, red-corner and blue-corner accent stripes at top and bottom,
premium matte finish, central area intentionally empty for competitor names and
score. No text, no logos, editorial sports-tech style.
```
Composited by `@bushi/rendering` (`resultCardHtml` / `leaderboardCardHtml` /
`certificateHtml`).

## Text prompts (Workers AI)

These mirror what's in the API and `@bushi/ai`'s `PROMPTS` registry. The
`/api/ai` routes already run promo copy and the assistant, logging every call to
`ai_generations` and falling back to a deterministic template when `AI` is
unbound.

### Tournament promo copy (`/api/ai/tournament/:id/promo`)
- **System:** _"You are a sharp martial-arts marketing copywriter. Tone:
  disciplined, premium, athletic. Avoid clichés and emoji."_
- **User:** _"Write a punchy 90-word promotional announcement for the tournament
  `{name}` on `{startDate}`[ in `{city}`]. Styles featured: `{styles}`. End with
  a call to register."_
- **Fallback** (deterministic): a templated announcement composed from name +
  style labels, so the UI always has copy.

### Organizer assistant (`/api/ai/assistant`)
- **System:** _"You are Bushi's organizer assistant. You help run martial arts
  tournaments: divisions, brackets, registrations, schedules, and reminders. Be
  concise and action-oriented."_
- **User:** the organizer's free-text message.

### Additional templates in `@bushi/ai`
`PROMPTS` also covers post-event recaps, school recaps, social captions, sponsor
thank-yous, an FAQ generator, and a coach assistant — each rendered via
`renderTemplate` (`{{var}}` substitution) and, for structured output,
`generateJSON<T>` with a safe fallback.

## How to regenerate & store in R2

Generated media belongs in the `GENERATED` R2 bucket (`bushi-generated`) and is
tracked in the `generated_assets` table (`kind`, `r2_key`, `width`, `height`,
`prompt`, `model`). The intended pipeline (Phase 1 in
[`roadmap.md`](./roadmap.md)):

1. Enqueue a `generate_asset` job (`{ assetKind, tournamentId, prompt }`) — the
   `JobMessage` shape already exists in `env.ts`.
2. The queue consumer calls `@bushi/ai` `generateImage(prompt, { seed })` (FLUX
   schnell) for backgrounds, then `@bushi/rendering` composites headline/score
   text over the background with Browser Rendering (`renderToImage` /
   `renderToPdf`).
3. `storeInR2(GENERATED, key, bytes)` writes the object; a `generated_assets` row
   records `kind`, `r2_key`, dimensions, the exact `prompt`, and the `model`.
4. Public pages reference the R2 object (e.g. OG image on a tournament page).

**Reproducibility:** use a **fixed seed per asset kind** (e.g. hero, og, result
card) so a regeneration produces the same base image; the composited text layer
is deterministic from the tournament/match data. Keep the prompt text in this doc
as the source of truth and update `generated_assets.prompt` whenever it changes,
so every stored asset is traceable back to the exact prompt + model that made it.

---

## Guardrails

- Never bake real logos or sponsor marks into a model prompt — composite those
  from `ASSETS` (uploaded logos) in the rendering layer.
- Never generate literal event text with the image model; headlines/dates/scores
  are always composited so they stay crisp and correct.
- Log every text generation to `ai_generations` (feature, model, prompt_key,
  latency, status) — as the `/api/ai` routes already do — for cost and quality
  observability.

## Cinematic hero video — "Blimp Cam" (Higgsfield)

A ~30s looping aerial drift across tournament floors, used full-bleed in the
homepage `SizzleReel` band. AI video models render short clips, so produce
**4–6 clips (~6–8s each), one per style**, all using the same slow aerial camera
move, then stitch into a seamless loop.

### Camera / look (keep consistent across every clip)
- **Camera move:** slow continuous aerial dolly + gentle pan — Higgsfield's
  "Bird's-eye / drone drift" or "crane push-in" preset. No cuts, no shake.
- **Altitude:** blimp-cam height — high enough to see the full mat + crowd, low
  enough to read athletes.
- **Grade:** premium sports-broadcast; restrained palette, cool neutrals with a
  single warm accent light. Sharp, clean, editorial. 24fps, cinematic.
- **Avoid:** dragons, flames, samurai cosplay, anime, neon, fantasy — nothing
  cliché. Real modern tournament venues only.

### Per-style prompts (one clip each)
1. **Karate** — "Slow aerial drone drift over a modern indoor karate tournament:
   multiple matted rings, athletes in white gi mid-kumite, referees, a full
   crowd in the stands under clean arena lighting. Premium sports-broadcast grade,
   restrained cool palette, cinematic 24fps, continuous smooth camera, no cuts."
2. **Brazilian Jiu-Jitsu** — "…over a BJJ competition: grapplers in gi and no-gi
   on blue and red mats, coaches cornering, packed bleachers, banners. Same slow
   aerial drift, premium broadcast look."
3. **Taekwondo** — "…over a taekwondo tournament: electronic hogu chest guards,
   fast spinning kicks, digital scoreboards, bright disciplined arena."
4. **Judo** — "…over a judo tournament: white and blue judogi, throws and mat
   work on a tatami, formal officials, tiered crowd."
5. **Kickboxing / amateur MMA** — "…over a striking event: a raised ring, wraps
   and gloves, cornermen, a dense engaged crowd, moody key light."
6. *(optional)* **Wide establisher** — "…highest altitude pass revealing the
   whole multi-mat venue at once, crowd and warmup areas visible."

### Format & delivery
- **Aspect:** 16:9, 1080p (or 4K downscaled). Generate a 9:16 variant later if we
  want a mobile-specific loop.
- **Stitch + loop + compress** (ffmpeg): crossfade the clips and mute audio —
  ```bash
  # concat with 0.5s crossfades, then compress for web, then make a poster frame
  ffmpeg -i c1.mp4 -i c2.mp4 -i c3.mp4 -i c4.mp4 -i c5.mp4 \
    -filter_complex "[0][1]xfade=transition=fade:duration=0.5:offset=6[a];\
    [a][2]xfade=transition=fade:duration=0.5:offset=11.5[b];\
    [b][3]xfade=transition=fade:duration=0.5:offset=17[c];\
    [c][4]xfade=transition=fade:duration=0.5:offset=22.5,format=yuv420p[v]" \
    -map "[v]" -an sizzle_raw.mp4
  ffmpeg -i sizzle_raw.mp4 -vf "scale=1920:-2" -c:v libx264 -crf 24 -preset slow \
    -movflags +faststart -an bushi-sizzle.mp4
  ffmpeg -i bushi-sizzle.mp4 -ss 2 -frames:v 1 bushi-sizzle-poster.jpg
  ```
- Aim for **< 8 MB**. Muted, looped, `playsInline` — handled by `SizzleReel`.

### Wiring it into the site
Host the file (see options in `deployment.md`), then set two build vars and
re-deploy:
- `VITE_HERO_VIDEO_URL` = public URL of `bushi-sizzle.mp4`
- `VITE_HERO_VIDEO_POSTER` = public URL of `bushi-sizzle-poster.jpg`
Until these are set, the band renders nothing (no layout impact).
